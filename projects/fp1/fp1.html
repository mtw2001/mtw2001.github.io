Development Process

I began by revisiting my previously created project that uses the Spotify API and began
converting this from a Python project to Javascript. Tackling this conversion proved to be
the bulk of this prototype. The Spotify API and specifically the oAuth 2.0 flow was even
trickier than I remembered. I had to rethink nearly every aspect of how the web app was
created so that it could work with Javascript. This took a lot of head scratching, Googling
and frustration. I came across a usefull tutorial, available here that helped quite
a bit with the oAuth problems I was facing.

I had originally planned to fully recreate my project in Javascript for the prototype,
but was unable to do so in a timely manner while balancing other classes and final projects
that were do this week. I think I tackled the most challenging part of the project though.

Issue Deep Dive

The main issue I ran into was one that I remembered from when I created the app in Python.
The Spotify API authorization process is very tricky and complicated. It requires a specific
redirect url when the user authenticates the app, which much also be added to a list on the
Spotify API dashboard. This puzzled be for quite some time until I remembered over a year
ago when I struggled with this same issue. I think my previous struggles are coming back
to me, so I am confident I will move more hastily towards my final submission.
